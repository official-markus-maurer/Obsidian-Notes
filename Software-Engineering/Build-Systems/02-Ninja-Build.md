# Ninja Build System

Ninja is a small build system with a focus on speed. It differs from Make in two major ways:
1.  It is designed to have its input files generated by a higher-level build system (like CMake or Meson).
2.  It is designed to run builds as fast as possible.

## Key Features
-   **Parallel by Default**: Automatically utilizes all CPU cores.
-   **Minimal Overhead**: Start-up time is almost zero.
-   **Simple Syntax**: The `build.ninja` file is machine-readable, not human-writable.

## Using Ninja with CMake
CMake can generate Ninja files instead of Makefiles.

```bash
# Generate build files for Ninja
cmake -G Ninja -B build .

# Build using Ninja
cmake --build build
# OR
ninja -C build
```

## Structure of `build.ninja` (For curiosity)
You rarely write this by hand, but here is what it looks like:

```ninja
cflags = -Wall -Wextra

rule cc
  command = gcc $cflags -c $in -o $out

build main.o: cc main.c
build utils.o: cc utils.c

build app: link main.o utils.o
  command = gcc $in -o $out
```

## Why it's faster
Make has to parse Makefiles, compute the DAG (Directed Acyclic Graph), and check file timestamps. For large projects (like Chrome or LLVM), this parsing takes seconds. Ninja does this nearly instantly because the dependency graph is pre-computed by the generator (CMake/Meson).
