# Software Design Patterns

Design patterns are reusable solutions to common problems. While often associated with OOP languages like Java, they are universal concepts.

## 📋 Topics

1.  **[[01-Singleton|Singleton]]**
    -   Ensuring a single instance of a resource.
    -   Global state management.

2.  **[[02-Factory|Factory]]**
    -   Creating objects without specifying the exact type.
    -   Decoupling creation from usage.

3.  **[[03-Observer|Observer]]**
    -   Event handling.
    -   Publish-Subscribe mechanism.
