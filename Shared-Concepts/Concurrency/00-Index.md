# Concurrency

Concurrency is the ability of different parts of a program to be executed out-of-order or in partial order, without affecting the final outcome.

## Key Concepts

1.  **[[01-Processes-vs-Threads|Processes vs Threads]]**
    -   Process: Isolated memory space.
    -   Thread: Shared memory space.
2.  **[[02-Synchronization|Synchronization]]**
    -   Mutexes (Mutual Exclusion).
    -   Semaphores (Signaling).
    -   Atomic Operations.
3.  **[[03-Deadlocks|Deadlocks]]**
    -   Causes and Prevention.
