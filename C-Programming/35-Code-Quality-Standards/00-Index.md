# Code Quality Standards

Writing C code that "works" is only the first step. In professional environments, especially in safety-critical systems (automotive, aerospace, medical), code must adhere to strict standards to ensure reliability, security, and maintainability.

## Topics
1.  [[01-MISRA-C]]: The gold standard for safety-critical C code.
2.  [[02-CERT-C]]: Security-focused coding standards.
3.  [[03-Style-Guides]]: Formatting, naming conventions, and readability.

## Why Standards Matter
-   **Reliability**: Reduces undefined behavior and bugs.
-   **Security**: Mitigates vulnerabilities like buffer overflows.
-   **Maintainability**: Ensures code is readable and consistent across teams.
-   **Portability**: Avoids compiler-specific extensions and implementation-defined behavior.
