# Continuous Integration (CI)

Modern C development relies on CI to ensure that code builds on all platforms and passes tests before merging.

## Topics
1.  [[01-GitHub-Actions]]: Setting up a CI workflow for C.
2.  [[02-Code-Coverage]]: Integrating `gcov` and uploading results.

## Why CI?
-   **Cross-Platform**: Verify your code works on Linux, Windows, and macOS automatically.
-   **Quality Gate**: Prevent broken code from entering the main branch.
